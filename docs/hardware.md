# Hardware and device contract

Hardware names are explicit configuration, not portable constants. Enumerate a
new host and update its ignored `.env` before starting the role.

## TurtleBot3 base, OpenCR, and LiDAR

- OpenCR defaults to `TB3_OPENCR_DEVICE=/dev/ttyACM0`.
- LDS-02 defaults to the stable link `TB3_LIDAR_DEVICE=/dev/tb3_lidar`; verify
  the udev rule resolves to the current USB adapter.
- `turtlebot3_bringup` must publish `/odom` and subscribe to the discovered base
  velocity topic before the application stack is accepted.
- `motion_controller_node` resolves the configured TB3 velocity topic and is the
  only application node allowed to publish velocity. VLM output never publishes
  `/cmd_vel` directly.
- The current lab contract uses `/robot_a/cmd_vel`. If a new ROBOTIS image uses
  `/cmd_vel` or a namespaced controller topic, set `TB3_CMD_VEL_TOPIC`, verify one
  subscriber, and run dry-run plus emergency-stop checks before real motion.

Useful read-only checks:

```bash
ls -l /dev/ttyACM0 /dev/tb3_lidar
ros2 topic info /odom
ros2 topic info /robot_a/cmd_vel
ros2 topic echo /odom --once
```

## Camera

- Default: `TB3_CAMERA_DEVICE=/dev/video0`.
- Enumerate with `v4l2-ctl --list-devices` and verify the selected node produces
  frames before launch.
- Container access requires the device mapping in the TB3 Compose template.
- A changed `/dev/videoN` index must be fixed in `.env`; do not edit the Python
  node or launch file on one host.

## Microphone and speaker

- Default capture: `TB3_MIC_ALSA_DEVICE=plughw:CARD=Device,DEV=0`.
- Default playback: `TB3_SPEAKER_ALSA_DEVICE=plughw:CARD=UACDemoV10,DEV=0`.
- Enumerate with `arecord -l` and `aplay -l`. USB card numbers can change after
  reboot; prefer stable ALSA card names where available.
- Validate capture with `scripts/smoke_microphone_host.sh` and playback through
  the no-motion TTS/speech smoke test. Avoid direct ALSA tests while the ROS
  capture/playback node owns the same device.

## Display and browser

- Default X display: `TB3_DISPLAY=:0`.
- Xorg, Openbox, iDesk, and Epiphany are owned by transient user services named
  `tb3-ui-*`. Epiphany is single-instance guarded and tied to the display stack
  lifecycle.
- The desktop shortcut is generated by `deploy/tb3/install.sh`; do not keep a
  host-only launcher as the sole startup path.
- Verify `http://127.0.0.1:$TB3_UI_PORT/state.json`, service status, and the
  visible page after a cold boot and SSH logout.

## Physical motion gate

Real motion requires all of the following: clear floor, charged base, reachable
hardware emergency stop, valid odometry, one velocity publisher path, reviewed
bounded plan, and an explicit `TB3_BEHAVIOR_DRY_RUN=false`. Repository and
clean-host validation never enable real motion automatically.
